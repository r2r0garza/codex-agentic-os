Heartbeat cadence, polling, and loop termination are driven by injectable `clock`/`sleeper`/`should_continue` parameters for deterministic tests; the default `sleeper` resolves `time.sleep` lazily inside the function body (not as a bound default parameter) specifically so `monkeypatch.setattr("codex_agentic_os.worker.time.sleep", ...)` can end an otherwise-infinite loop in tests. Approval-required and unresolved-context-reference dispatch errors intentionally propagate unchanged — skip/idle policy for those is #83's scope.
